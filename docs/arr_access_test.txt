1. Valid Read Access (Rvalue / Movable Value)
First Element: [10, 20, 30][0] -> Expect 10.

Middle Element: [10, 20, 30][1] -> Expect 20.

Last Element: [10, 20, 30][2] -> Expect 30.

Nested Read: [[10, 20], [30, 40]][1][0] -> Expect 30.

Move Semantics Verification: If your language/evaluator tracks moved states, test let x = arr[1] (where arr[1] is a movable type, like a nested array or string). Verify that x takes ownership of the data and that arr[1] is left in a valid, empty "moved-from" state (if applicable to your architecture).

2. Valid Write Access (Lvalue / Reference Assignment)
Modify First Element: arr = [1, 2, 3]; arr[0] = 99; -> Expect arr to evaluate to [99, 2, 3].

Modify Last Element: arr = [1, 2, 3]; arr[2] = 99; -> Expect arr to evaluate to [1, 2, 99].

Chained Assignment: arr[0] = arr[1] = 99; -> Expect indices 0 and 1 to both equal 99.

Nested Write: arr = [[1, 2]]; arr[0][1] = 99; -> Expect arr to evaluate to [[1, 99]].

Type Mutation (if dynamically typed): arr = [1, 2]; arr[0] = "test"; -> Expect arr to evaluate to ["test", 2].

Self-Assignment: arr[0] = arr[0]; -> Verify the evaluator doesn't clear the memory during the lvalue-to-rvalue resolution before writing.

3. Out of Bounds & Negative Indices (Exceptions)
All of these must throw an exception indicating an out-of-bounds error.

Negative Index (-1): [1, 2, 3][-1] -> Throws.

Extreme Negative Index: [1, 2, 3][INT_MIN] -> Throws.

Index Exactly at Length: [1, 2, 3][3] -> Throws (0-indexed arrays mean 3 is out of bounds).

Index Beyond Length: [1, 2, 3][5] -> Throws.

Extreme Positive Index: [1, 2, 3][INT_MAX] -> Throws.

Empty Array Access: [][0] -> Throws.

4. Index Type Validation (Exceptions)
The specification dictates that ONLY int is allowed as an index. All of these must throw an exception regarding invalid index type.

Float Index (Whole Number): [1, 2, 3][1.0f] -> Throws. (Crucial: Do not silently cast 1.0 to 1).

Float Index (Fractional): [1, 2, 3][1.5f] -> Throws.

String Index: [1, 2, 3]["1"] -> Throws.

Boolean Index: [1, 2, 3][true] -> Throws. (Crucial: Do not silently cast true to 1 or false to 0).

Array Index: [1, 2, 3][[1]] -> Throws.

5. Base Object Type Validation (Exceptions)
The specification dictates the operator fails if not done on an array. All of these must throw an exception regarding invalid base type.

Integer Base: 5[0] -> Throws.

Float Base: 5.5f[0] -> Throws.

Boolean Base: true[0] -> Throws.

String Base: "hello"[0] -> Throws. (Unless your language intentionally treats strings as character arrays under the hood; if strict array types are expected, this must fail).
