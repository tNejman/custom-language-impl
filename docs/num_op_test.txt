1. Integer Operations (int, int)
Addition (NumericOp::ADD)

Standard Positive: 5 + 10 -> Expect 15.

Standard Negative: -5 + (-10) -> Expect -15.

Zero Identity: 0 + 0 -> Expect 0; 10 + 0 -> Expect 10.

Positive Overflow (Clamping): INT_MAX + 1 -> Expect INT_MAX.

Extreme Positive Overflow: INT_MAX + INT_MAX -> Expect INT_MAX.

Negative Overflow (Clamping): INT_MIN + (-1) -> Expect INT_MIN.

Extreme Negative Overflow: INT_MIN + INT_MIN -> Expect INT_MIN.

Subtraction (NumericOp::SUB)

Standard Positive: 10 - 5 -> Expect 5.

Standard Negative: -10 - (-5) -> Expect -5.

Zero Identity: 10 - 0 -> Expect 10; 0 - 10 -> Expect -10.

Positive Overflow (Clamping): 5 - INT_MIN -> Expect INT_MAX.

Extreme Positive Overflow: INT_MAX - INT_MIN -> Expect INT_MAX.

Negative Overflow (Clamping): -5 - INT_MAX -> Expect INT_MIN.

Extreme Negative Overflow: INT_MIN - INT_MAX -> Expect INT_MIN.

Multiplication (NumericOp::MUL)

Standard Positive: 5 * 10 -> Expect 50.

Standard Negative: -5 * 10 -> Expect -50; -5 * -10 -> Expect 50.

Zero Identity: 10 * 0 -> Expect 0.

Positive Overflow (Clamping): (INT_MAX / 2 + 1) * 2 -> Expect INT_MAX.

Extreme Positive Overflow: INT_MAX * INT_MAX -> Expect INT_MAX.

Negative Overflow (Clamping): (INT_MAX / 2 + 1) * -2 -> Expect INT_MIN.

Extreme Negative Overflow: INT_MAX * INT_MIN -> Expect INT_MIN.

Division (NumericOp::DIV)

Any Operation: 10 / 2, 1 / 0, 0 / 0 -> ALL must throw InvalidOperationException with the message "cannot divide ints" (and verify Position{1, 1}). Your code universally rejects integer division.

Modulo (NumericOp::MOD)

Standard Positive: 10 % 3 -> Expect 1.

Standard Negative: -10 % 3 -> Expect -1 (C++ standard behavior).

Negative Divisor: 10 % -3 -> Expect 1.

Zero Divisor (Exception): 10 % 0 -> Must throw InvalidOperationException with message "cannot mod by 0".

Edge Case (INT_MIN % -1): INT_MIN % -1 -> Expect 0. Note: In standard C++, doing this directly on 32-bit ints causes undefined behavior/hardware faults. Your code casts to long long first, safely bypassing this hardware trap, making this an excellent correctness test.

2. Floating-Point Operations (float, float)
Note: These tests assume MAX_FLOAT and MIN_FLOAT are defined constants in your scope. You will need to test values that exceed these bounds.

Addition (NumericOp::ADD)

Standard Positive: 5.5f + 2.2f -> Expect 7.7f (accounting for floating-point epsilon).

Standard Negative: -5.5f + -2.2f -> Expect -7.7f.

Positive Overflow (Clamping): MAX_FLOAT + 100.0f (or equivalent magnitude) -> Expect MAX_FLOAT.

Negative Overflow (Clamping): MIN_FLOAT + (-100.0f) -> Expect MIN_FLOAT.

Subtraction (NumericOp::SUB)

Standard Operation: 5.5f - 2.2f -> Expect 3.3f.

Positive Overflow (Clamping): (MAX_FLOAT / 2) - MIN_FLOAT -> Expect MAX_FLOAT.

Negative Overflow (Clamping): (MIN_FLOAT / 2) - MAX_FLOAT -> Expect MIN_FLOAT.

Multiplication (NumericOp::MUL)

Standard Operation: 2.5f * 4.0f -> Expect 10.0f.

Multiplication by Zero: 10.5f * 0.0f -> Expect 0.0f.

Positive Overflow (Clamping): MAX_FLOAT * 2.0f -> Expect MAX_FLOAT.

Negative Overflow (Clamping): MAX_FLOAT * -2.0f -> Expect MIN_FLOAT.

Division (NumericOp::DIV)

Standard Operation: 10.0f / 2.5f -> Expect 4.0f.

Zero Divisor (Exception): 10.0f / 0.0f -> Must throw InvalidOperationException with message "cannot divide by 0".

Positive Overflow (Clamping): MAX_FLOAT / 0.5f (equivalent to multiplying by 2) -> Expect MAX_FLOAT.

Negative Overflow (Clamping): MIN_FLOAT / 0.5f -> Expect MIN_FLOAT.

Modulo (NumericOp::MOD)

Standard Operation: 5.5f % 2.0f -> Expect 1.5f.

Zero Divisor (Exception): 5.5f % 0.0f -> Must throw InvalidOperationException with message "cannot mod by 0".

Negative Modulo: -5.5f % 2.0f -> Verify behavior matches your FmodModulus{} implementation (usually -1.5f via std::fmod).

3. Type Mismatches and Invalid Types
These tests target the fallback lambda []( const auto&, const auto& ) at the end of your std::visit. Every case here should throw an InvalidOperationException with the message "numeric operations are only legal on 'int-int' and 'float-float' pairs".

Mixed Types (Int LHS, Float RHS): int(5) + float(5.5).

Mixed Types (Float LHS, Int RHS): float(5.5) + int(5).
